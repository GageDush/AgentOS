# Claude Design Prompt

Design an AgentOS dashboard/base application shell using the attached `design.md` and `wireframe.md`.

Scope is dashboard-first.

Use the included scraper `assets/`, `screenshots/`, and `raw/` files only as visual reference for polished website craft, motion feel, section rhythm, and asset density. Do not copy proprietary generated code or treat the scraped source brand as AgentOS branding.

The first screen should be the working AgentOS dashboard, not a marketing landing page.

Build a calm, premium operator interface with:

- persistent top navigation
- compact runtime health strip
- active mission panel
- pending approvals panel
- runtime posture panel
- activity timeline
- agent/session panel
- right-side inspector rail on desktop
- stacked mobile layout
- intentional empty states
- subtle dashboard motion

Use straightforward product naming:

- Dashboard
- Missions
- Approvals
- Activity
- Agents
- Archive
- Wiki
- Loadout
- Settings
- Scraper

Avoid military, spy, or secret-ops language.

The result should feel like a serious local-first agent operations dashboard for a solo developer or small technical team.
